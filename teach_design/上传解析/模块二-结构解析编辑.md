# 模块二：结构解析编辑

## 1. 概况

本模块提供对已解析文档的章节结构进行查看和编辑的能力。文档上传解析完成后，用户可通过编辑器页面对 `contract_structure` 章节树进行审查和修正——包括修改章节编号与标题、编辑正文内容、增删节点、调整层级结构，修改结果保存回 `document.json`。

**核心能力**：
- 章节树读取与展示
- 节点编辑（label / title / content 正文段落）
- 结构增删（添加子节点/兄弟节点、删除节点及级联子节点）
- 树完整性校验（node_id 唯一性、parent_id 引用有效、无循环引用）
- 原子保存（仅替换 contract_structure 字段，保留 texts/tables 等不变）

## 3. 设计

### 3.1 读取与展示

- **文档信息接口** 返回 `schema_name`、`chapter_count`、`warning_count`、`origin` 等概要，用于顶部工具栏展示
- **结构接口** 返回完整的 `contract_structure` 树（含 ROOT 节点），前端递归渲染为可折叠树形视图
- 每个树节点展示：编号(label) + 标题(title) + 类型标签(kind) + 页码(page_no) + 告警指示(warnings)

### 3.2 编辑与提交

前端采用**双面板布局**：左侧章节树、右侧编辑面板。修改在内存中操作，点击「保存到服务器」时提交完整树。

**可编辑字段**：
- `label` — 编号文本（如"第一章"、"1.1"）
- `title` — 标题正文
- `content[]` — 直属正文段落（每段独立 textarea）
- 节点增删 — 添加子节点/兄弟节点、删除节点（需确认，级联删除子节点）

**不可编辑字段**（只读展示）：
- `kind`、`page_no`、`source_ref`、`node_id`、`parent_id`（由系统管理）

### 3.3 后端校验规则

保存时 `_validate_structure_tree` 执行以下检查：

| 规则 | 说明 |
|------|------|
| node 类型 | structure 必须是 dict，children 必须是 list |
| node_id 必填 | 每个节点必须有非空 node_id |
| node_id 唯一 | 树中不能有重复的 node_id |
| parent_id 有效 | 非 root 节点的 parent_id 必须在树中存在 |
| 无循环引用 | 沿 parent_id 链追溯不能出现重复节点 |
| root 校验 | root 节点的 node_id 必须是 "root"，parent_id 必须为空 |

### 3.4 原子保存

保存时读取原 `document.json`，**仅替换** `contract_structure` 字段，保留所有其他字段（`texts`、`tables`、`warnings`、`schema_name` 等）不变。写入使用临时文件 `.json.part` → `rename` 原子替换。

## 4. 接口

### 4.1 获取文档信息

```
GET /api/v1/documents/{file_id}
```

**响应 200**：
```json
{
  "success": true,
  "data": {
    "document": {
      "file_id": "uuid",
      "schema_name": "DoclingDocument",
      "origin": { "filename": "合同.pdf", "mimetype": "application/pdf" },
      "has_structure": true,
      "chapter_count": 5,
      "warning_count": 112
    }
  }
}
```

**错误**：404 — `DOCUMENT_NOT_FOUND`

### 4.2 获取章节结构树

```
GET /api/v1/documents/{file_id}/structure
```

**响应 200**：
```json
{
  "success": true,
  "data": {
    "meta": { "schema_name": "DoclingDocument", "file_id": "uuid" },
    "structure": {
      "node_id": "root",
      "label": "",
      "title": "ROOT",
      "kind": "root",
      "children": [
        {
          "node_id": "node_0001",
          "parent_id": "root",
          "label": "第一章",
          "title": "协议总则",
          "kind": "chapter",
          "rank": 0,
          "page_no": 3,
          "source_ref": "#/texts/9",
          "content": [
            { "source_ref": "#/texts/10", "text": "本章约定了..." }
          ],
          "warnings": [],
          "children": [ ... ]
        }
      ]
    }
  }
}
```

**错误**：404 — `DOCUMENT_NOT_FOUND`；422 — `STRUCTURE_VALIDATION_ERROR`（缺 contract_structure）

### 4.3 保存章节结构

```
PUT /api/v1/documents/{file_id}/structure
Content-Type: application/json
```

**输入**：
```json
{
  "structure": { ... }
}
```

`structure` 为完整的 contract_structure 树（含 ROOT 节点）。

**响应 200**：
```json
{
  "success": true,
  "message": "结构已保存"
}
```

**错误**：
| HTTP | code | 场景 |
|------|------|------|
| 404 | DOCUMENT_NOT_FOUND | 文档不存在 |
| 422 | STRUCTURE_VALIDATION_ERROR | 校验失败（含具体描述） |

### 4.4 编辑器页面

```
GET /editor/{file_id}
```

返回服务端渲染的编辑器 HTML 页面（非 API 接口）。

## 5. 数据

### 5.1 数据来源与存储

```
storage/parsing/{file_id}/document.json    ← 读写目标
```

- **读取**：直接从 `document.json` 中提取 `contract_structure` 字段
- **保存**：读取完整 `document.json` → 替换 `contract_structure` → 原子写回

### 5.2 关键字段说明

结构中 `source_ref` 因解析器不同有三种格式，**编辑和保存时必须原样保留**：

| 解析器 | source_ref 格式 | 示例 |
|--------|----------------|------|
| python-docx | `#/texts/{index}` | `#/texts/13` |
| docling (PDF) | `#/texts/{index}` | `#/texts/9` |
| pymupdf4llm | `#/pages/{chunk}/lines/{line}` | `#/pages/3/lines/0` |

### 5.3 节点字段全表

| 字段 | 类型 | 编辑性 | 说明 |
|------|------|--------|------|
| node_id | str | 只读 | 节点唯一标识 |
| parent_id | str | 系统管理 | 父节点 ID |
| label | str | 可编辑 | 编号标签（"第一章"、"1.1"） |
| title | str | 可编辑 | 标题正文 |
| kind | str | 只读 | 编号类型 |
| rank | int | 只读 | 层级权重 |
| path | list[int] | 只读 | 数字编号路径 |
| source_ref | str | 只读 | 原文溯源引用 |
| page_no | int | 只读 | PDF 页码 |
| content | list[dict] | 可编辑 | 直属正文段落 |
| tables | list[dict] | 只读 | 归属表格 |
| warnings | list | 只读 | 解析告警 |
| children | list[node] | 系统管理 | 子节点（递归） |
