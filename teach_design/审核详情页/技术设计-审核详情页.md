**版本**: v1.0  
**日期**: 2026-06-25  
**依赖文档**: 审核详情页技术设计.md（v2.0）、配置中心技术设计.md（v3.0）

---

## 一、概要

### 1.1 背景

条款召回（docling 解析 blocks.json + BM25/向量检索）已作为前置能力实现。本设计专注于：

1. **复用召回结果**：将召回层返回的 `hit_block_no` + `bbox` 衔接写入 `audit_result_item`，供详情页定位高亮。
2. **按维度 + 风险点展示**：右栏按维度 Tab 分组、风险等级过滤，每条风险点独立一张卡片铺平（与截图一致）。

### 1.2 核心决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| 召回层复用方式 | 召回服务返回 `{block_no, score}`，AI 引擎选取 top-1 写入 hit_block_no | 召回与生成解耦，bbox 由后端按 block_no 回查 blocks.json 填充 |
| 结果展示粒度 | 每个风险点一张卡片铺平，不再按审查点分组 | 与原型截图一致，信息更紧凑 |
| show/display 字段 | 保留 v2.0 设计：show=重大风险默认展开，display=是否被用户忽略 | 前端直接用，无额外状态管理 |
| 维度 Tab 计数 | 来自 `audit_dimension_stat`，审核写入阶段预聚合 | 避免实时 COUNT，Tab 切换无额外查询 |

## 三、召回结果复用设计

### 3.1 召回层输出接口（已实现，本期复用）

召回服务接受 `audit_point_id` + `blocks`，返回命中块列表：

```python
# 召回层已有实现，本期直接调用
def recall(audit_point: AuditPoint, blocks: list[Block]) -> list[RecallHit]:
    # BM25 + 向量混合检索
    # 返回按 score 降序的命中块
    ...

class RecallHit:
    block_no: int
    score: float
    text: str
```

### 3.2 AI 引擎衔接点（本期新增）

**流程**：

```
1. 读取 blocks.json
2. 对每个 audit_point 调用 recall() → 得到 top-K hit blocks
3. 将 top-K blocks 的 text 注入 LLM Prompt（含 block_no 标注）
4. LLM 返回 hit_block_no（模型从上下文中选取最相关的 block_no）
5. 后端按 hit_block_no 回查 blocks.json 取 bbox
6. 写入 audit_result_item
```

**Prompt 中 block 注入格式**：

```
相关合同条款：
[block_no=12] 若甲方在十个工作日内未提出书面异议，则视为项目自动验收通过。
[block_no=13] 乙方应提供验收测试报告及操作手册。

请基于以上条款判断风险，并在输出中指定 hit_block_no。
```

**LLM 输出格式**（沿用 v2.0，无变化）：

```json
{
  "title": "默示验收风险",
  "risk_level": 1,
  "risk_summary": "甲方未在期限内反馈即视为验收通过",
  "modify_suggestion": ["删除自动验收表述，改为双方签署书面验收单"],
  "clause_name": "验收条款",
  "hit_text": "若甲方在十个工作日内未提出书面异议，则视为项目自动验收通过。",
  "hit_block_no": 12
}
```

**bbox 回查逻辑**（伪代码）：

```python
def fill_bbox(hit_block_no: int, blocks_json: dict) -> dict:
    block = next((b for b in blocks_json["blocks"] if b["no"] == hit_block_no), None)
    if block is None:
        return {}
    return {
        "hit_page": block["page"],
        "hit_bbox_x": block["bbox"]["x"],
        "hit_bbox_y": block["bbox"]["y"],
        "hit_bbox_w": block["bbox"]["width"],
        "hit_bbox_h": block["bbox"]["height"],
    }
```

> `dim_id` / `dim_name` 由引擎按当前批次填入，LLM 无需感知。

## 四、DB 设计（沿用 v2.0，本节仅说明关键字段用途）

### 4.1 `audit_result_item` 关键字段说明

| 字段 | 来源 | 用途 |
|------|------|------|
| `dim_id` / `dim_name` | 引擎按批次填入 | 维度 Tab 分组 |
| `audit_point_id` | 召回 + LLM 批次 | 可追溯到配置中心审查点 |
| `risk_level` | LLM 输出 | 1=重大风险，2=一般风险 |
| `hit_block_no` | LLM 从召回 top-K 中选取 | 回查 bbox 用 |
| `hit_page` / `hit_bbox_*` | 引擎回查 blocks.json 填充 | 前端 pdf.js 高亮定位 |
| `show` | 接口计算：risk_level=1 → true | 前端卡片展开/收起初始状态 |
| `display` | 默认 1，用户忽略后置 0 | 是否在列表中显示 |

> `audit_result_item` 中 **每行对应一个风险点**（非审查点），与前端一张卡片一一对应。  
> 一个 `audit_point` 可以产出多行 `audit_result_item`（每个风险点一行）。

### 4.2 `audit_dimension_stat` 写入时机

每批维度 LLM 调用完成后即时写入/更新该维度的统计，前端 Tab 计数直接读此表。

```sql
INSERT INTO audit_dimension_stat (task_id, dim_id, dim_name, total_count, major_count, general_count)
VALUES (?, ?, ?, ?, ?, ?)
ON DUPLICATE KEY UPDATE
  total_count = VALUES(total_count),
  major_count = VALUES(major_count),
  general_count = VALUES(general_count);
```

## 六、前端展示设计

### 6.1 右栏交互流程

```
页面加载
  ├─ GET /detail        → 顶部合同信息 + 维度 Tab（含计数徽标）
  ├─ GET /pdf-url       → pdf.js 加载左侧 PDF
  └─ GET /results?dimId={第一个维度id}  → 渲染卡片列表

切换维度 Tab
  └─ GET /results?dimId={X}

切换风险等级过滤
  └─ GET /results?dimId={X}&level={1|2}

点击卡片「定位原文」
  └─ pdf.js.scrollPageIntoView({pageNumber: hitPage})
     → canvas 绘制高亮矩形（hitBbox × viewport.scale）

展开/收起修改建议
  └─ 本地切换 show（不请求后端）

忽略本条
  └─ PATCH .../display {display: 0}
     → 从列表移除该卡片
     → 更新 Tab 计数（本地 -1）
```

### 6.2 卡片结构（对应截图）

```
┌─────────────────────────────────────────────┐
│ ▼ 主体信息缺失风险           [重大风险]  [忽略] │
│                                              │
│ 风险描述                                     │
│ 甲方名称、地址、法定代表人均缺失...            │
│                                              │
│ 修改方案（show=true 时展开）                  │
│ • 补全甲方名称、注册地址...                   │
│ • 明确双方签章位置...                        │
│                                              │
│ [定位原文]  [查看条款]                        │
└─────────────────────────────────────────────┘
```

- `risk_level=1` → 红色标签「重大风险」，`show=true` 默认展开修改方案
- `risk_level=2` → 橙色标签「一般风险」，`show=false` 默认收起
- 「定位原文」触发 pdf.js 跳转 + 高亮
- 「忽略」调用 PATCH display=0，前端本地移除卡片并更新 Tab 计数徽标

### 6.3 维度 Tab 徽标格式

```
基础核查 (2)  法务风险 (2)  财务风险 (1)  形式核查 (1)
```

括号内数字来自 `audit_dimension_stat.total_count`（`display=1` 的）。
用户忽略后前端本地减 1 同步更新，无需重新请求 `/detail`。

## 七、性能与边界说明

| 场景 | 处理方式 |
|------|----------|
| 某维度无命中风险 | `audit_dimension_stat` 该维度不写入，`/detail` 的 dimensions 数组不含该维度，Tab 不展示 |
| `hit_block_no` LLM 未返回 | `hit_page=null, hitBbox=null`，卡片不展示「定位原文」按钮 |
| `hit_block_no` 回查不到 bbox | 同上，bbox 字段均为 null |
| 用户全部忽略某维度 | Tab 计数变为 0，仍显示 Tab，传 `showIgnored=true` 可恢复查看 |
| 任务尚未完成访问 /results | 返回 `{ code: 42201, msg: "任务尚未完成" }`，前端展示 loading 状态 |
