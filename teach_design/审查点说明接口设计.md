# 审查点说明接口设计

**版本**: v1.0  
**日期**: 2026-06-25

---

## 背景

合同审核流程中，用户需要了解当前合同类型下启用了哪些审查点、每个审查点的审查说明和风险点标准，以便理解审核结果的判断依据。

---

## 接口设计

### GET /api/v1/audit/contract-types/:contractTypeId/audit-points

获取指定合同类型下的审查点列表及其说明，按维度分组返回。

**Path 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| contractTypeId | number | 是 | 合同类型 ID（来自配置中心 contract_type.id） |

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| dimId | number | 否 | 按维度过滤，不传返回全部维度 |

**数据来源**：

```sql
-- 按合同类型查询启用的审查点（双层 enabled 过滤），按维度分组
SELECT
  d.id AS dim_id, d.name AS dim_name, d.sort_order AS dim_sort,
  ap.id, ap.name, ap.description, ap.instruction, ap.risk_points, ap.sort_order
FROM contract_type_audit_point ctap
JOIN audit_point ap ON ap.id = ctap.audit_point_id
JOIN dimension d ON d.id = ap.dim_id
WHERE ctap.contract_type_id = ?
  AND ctap.enabled = 1
  AND ap.enabled = 1
  AND ap.deleted_at IS NULL
ORDER BY d.sort_order, ap.sort_order;
```

后端按 `dim_id` 聚合分组后返回嵌套结构，前端无需二次处理。

**错误响应**：

| code | HTTP | 说明 |
|------|------|------|
| 0 | 200 | 成功 |
| 40401 | 404 | 合同类型不存在或已停用 |
| 50001 | 500 | 内部错误 |

## 使用场景

1. **审核前预览**：用户发起审核前，可查看该合同类型下共有哪些审查点及判断标准
2. **审核结果辅助说明**：详情页"已选 N 个审核点"标签点击后展开说明
3. **配置确认**：管理员确认某合同类型配置是否正确
