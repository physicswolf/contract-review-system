# 模块三：合同任务列表

## 1. 概况

本模块提供已上传合同的集中管理和检索能力。每当一份合同上传并解析完成后，系统自动从 `document.json` 中抽取关键元信息（合同名称、甲乙方、合同类型）写入 MySQL 数据库。用户可通过合同列表页面进行多条件检索、查看审核详情（预留）和删除合同（含关联文件清理）。

**核心能力**：
- 解析完成时自动抽取合同元信息并入库
- 合同列表多条件模糊检索（合同名称、甲方、乙方、合同类型）
- 合同 CRUD（手动创建、查询、更新、删除）
- 删除合同时级联清理上传文件、元数据和解析目录
- 审核详情预留接口

## 3. 设计

### 3.1 仓库模式

采用抽象接口 + 具体实现的仓库模式，当前使用 MySQL（`pymysql`），后续可替换为 PostgreSQL 等：

```
ContractRepository (抽象接口)
  · initialize()     建库建表
  · insert()         插入记录
  · upsert_by_file_id()  按file_id插入或更新
  · find_all()       多条件查询
  · find_by_id()     单条查询
  · update_by_id()   按id更新
  · delete_by_id()   按id删除
  · count()          计数
        │
        └── MysqlContractRepository (MySQL 实现)
              · pymysql.connect()
              · DictCursor（字典返回）
              · 自动建库 CREATE DATABASE IF NOT EXISTS
              · 自动建表 CREATE TABLE IF NOT EXISTS
```

### 3.2 元信息自动抽取

解析成功后自动执行（`document_tasks.py` 中 hook），抽取失败不影响解析成功状态：

| 字段 | 来源 | 方法 |
|------|------|------|
| contract_name | `origin.filename` | 去扩展名，归一化空格 |
| party_a | `contract_structure` 全文 | 正则 `甲方[：:]\s*(.{2,30}?(公司\|集团\|中心...))` |
| party_b | 同上 | 正则 `乙方[：:]\s*(.{2,30}?(公司\|集团\|中心...))` |
| contract_type | `filename` | 关键词匹配（采购/服务/租赁/合作/技术/保密/劳动），默认"未分类" |
| file_id | 上传 UUID | 直接赋值 |
| review_time | 当前时间 | 上海时区 ISO8601 |

### 3.3 删除级联清理

删除合同记录时同步清理三个位置：

```
1. MySQL: DELETE FROM contracts WHERE id = :id
2. uploads: 遍历 storage/uploads/**/{file_id}.*  → unlink
3. parsing: rmtree storage/parsing/{file_id}/
```

所有文件操作使用 `missing_ok=True` / `ignore_errors=True`，确保幂等安全。

### 3.4 搜索筛选

前端四个输入框（合同名称/甲方/乙方/合同类型），均支持 LIKE 模糊匹配，300ms 防抖自动搜索。重置按钮清空条件并恢复全量列表。

## 4. 接口

### 4.1 查询合同列表

```
GET /api/v1/contracts?contract_name=采购&party_a=华城&party_b=&contract_type=
```

**Query 参数**（均为可选，空值不过滤）：

| 参数 | 类型 | 说明 |
|------|------|------|
| contract_name | string | 合同名称模糊搜索 |
| party_a | string | 甲方模糊搜索 |
| party_b | string | 乙方模糊搜索 |
| contract_type | string | 合同类型模糊搜索 |

**响应 200**：
```json
{
  "success": true,
  "data": {
    "contracts": [
      {
        "id": 1,
        "file_id": "8d68b3ee-5392-435b-a26a-f10aab24ecfe",
        "contract_name": "智慧园区软件采购合同",
        "party_a": "华城数字建设有限公司",
        "party_b": "云启科技有限公司",
        "contract_type": "采购合同",
        "review_time": "2026-06-25T10:20:00+08:00",
        "created_at": "2026-06-25T10:20:00+08:00"
      }
    ],
    "total": 1
  }
}
```

### 4.2 创建合同

```
POST /api/v1/contracts
Content-Type: application/json
```

**输入**：
```json
{
  "file_id": "uuid（必填）",
  "contract_name": "合同名称",
  "party_a": "甲方名称",
  "party_b": "乙方名称",
  "contract_type": "采购合同",
  "review_time": "2026-06-25T10:00:00+08:00"
}
```

**响应 201**：
```json
{
  "success": true,
  "data": { "contract": { "id": 2, ... } },
  "message": "合同记录已创建"
}
```

### 4.3 获取合同详情

```
GET /api/v1/contracts/{contract_id}
```

**响应 200**：
```json
{
  "success": true,
  "data": {
    "contract": {
      "id": 1,
      "file_id": "uuid",
      ...
      "detail_url": null
    }
  }
}
```

`detail_url` 当前固定为 `null`，为后续审核详情页预留。

**错误**：404 — `CONTRACT_NOT_FOUND`

### 4.4 更新合同信息

```
PUT /api/v1/contracts/{contract_id}
Content-Type: application/json
```

**输入**（所有字段可选，仅更新传入的非 null 字段）：
```json
{
  "contract_name": "新名称",
  "party_a": "新甲方",
  "party_b": null,
  "contract_type": null
}
```

**响应 200**：
```json
{
  "success": true,
  "data": { "contract": { ... } },
  "message": "合同信息已更新"
}
```

### 4.5 删除合同

```
DELETE /api/v1/contracts/{contract_id}
```

**响应 200**：
```json
{
  "success": true,
  "message": "合同及相关文件已删除"
}
```

删除行为：先删 MySQL 记录，再清理 `uploads` 文件，最后删 `parsing` 目录。文件不存在时报错被吞掉（幂等）。

**错误**：404 — `CONTRACT_NOT_FOUND`

### 4.6 合同列表页面

```
GET /contracts
```

返回服务端渲染的合同列表 HTML 页面（非 API 接口）。

## 5. 数据

### 5.1 MySQL 数据库

- **数据库名**：`aizhiqi`
- **字符集**：`utf8mb4`，排序 `utf8mb4_unicode_ci`
- **引擎**：InnoDB

### 5.2 表结构

```sql
CREATE TABLE contracts (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    file_id         VARCHAR(64)  NOT NULL,
    contract_name   VARCHAR(256) NOT NULL DEFAULT '',
    party_a         VARCHAR(256) NOT NULL DEFAULT '',
    party_b         VARCHAR(256) NOT NULL DEFAULT '',
    contract_type   VARCHAR(64)  NOT NULL DEFAULT '未分类',
    review_time     VARCHAR(32)  NOT NULL,
    created_at      VARCHAR(32)  NOT NULL,
    updated_at      VARCHAR(32)  NOT NULL,
    UNIQUE KEY uk_file_id (file_id),
    INDEX idx_review_time (review_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 5.3 连接配置

通过 `.env` 文件配置，`pydantic-settings` 自动加载：

```bash
MYSQL_HOST=192.168.1.155
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=*
MYSQL_DATABASE=aizhiqi
```

`Settings` 类中对应字段：`mysql_host`、`mysql_port`、`mysql_user`、`mysql_password`、`mysql_database`。

### 5.4 关联文件存储

合同记录的 `file_id` 关联以下磁盘文件：

```
storage/uploads/YYYY/MM/DD/{file_id}.pdf     ← 原始上传文件
storage/uploads/YYYY/MM/DD/{file_id}.json    ← 上传元数据
storage/parsing/{file_id}/document.json      ← 解析产物
storage/parsing/{file_id}/document.pdf       ← DOCX转换PDF（可选）
```
