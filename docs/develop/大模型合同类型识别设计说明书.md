## 一、新建 `services/llm_classifier.py`

### 1.1 模块职责

- 从解析后的文档提取合同首部正文
- 查询已启用合同类型清单
- 构造 prompt 并调用 LLM
- 解析 LLM 返回结果
- 所有异常降级返回 `None`，由调用方走原有关键词规则

### 1.2 核心函数

```python
import httpx
import json
import logging
from src.config import get_settings
from src.services.structure_editor import load_document_json
from src.services.contract_type_store import contract_type_repository

_logger = logging.getLogger(__name__)


def classify_contract_type(file_id: str) -> tuple[str, int] | None:
    """调用大模型推断合同类型。
    返回 (类型名称, 置信度百分比) 或 None（降级）。"""
    settings = get_settings()
    if not settings.llm_api_url:
        return None

    # 1. 提取合同首部正文
    preamble_text = _extract_preamble(file_id, settings)
    if not preamble_text:
        return None

    # 2. 查询已启用合同类型清单
    type_names = _enabled_type_names()
    if len(type_names) < 2:
        return None  # 少于2个类型时无需 LLM 判断

    # 3. 构造 prompt 并调用 LLM
    prompt = _build_prompt(preamble_text, type_names)
    result = _call_llm(prompt, settings)
    if result is None:
        return None

    # 4. 校验返回值
    result = result.strip()
    for name in type_names:
        if name in result:
            return name, _confidence_from_response(result, name)

    return None


def _extract_preamble(file_id: str, settings) -> str:
    """从 document.json 提取合同首部正文。"""
    try:
        document = load_document_json(file_id, settings)
    except Exception:
        return ""

    structure = document.get("contract_structure")
    if not isinstance(structure, dict):
        return ""

    texts = []
    # 优先取 preamble 节点（合同首部）
    for child in structure.get("children") or []:
        if isinstance(child, dict) and child.get("kind") == "preamble":
            for item in (child.get("content") or []):
                if isinstance(item, dict):
                    text = str(item.get("text") or "").strip()
                    if text:
                        texts.append(text)
            break

    # 没有 preamble 节点，取根节点直属正文
    if not texts:
        for item in (structure.get("content") or []):
            if isinstance(item, dict):
                text = str(item.get("text") or "").strip()
                if text:
                    texts.append(text)

    return "\n".join(texts[:30])  # 限制长度


def _enabled_type_names() -> list[str]:
    """查询所有已启用合同类型名称。"""
    try:
        records = contract_type_repository.find_all({"enabled": 1})
        return [r["name"] for r in records if r.get("name")]
    except Exception:
        return []


def _build_prompt(preamble: str, names: list[str]) -> str:
    name_list = "\n".join(f"- {n}" for n in names)
    return (
        "你是一个合同类型识别助手。请根据以下合同首部内容，判断该合同属于哪种类型。\n\n"
        f"可选合同类型清单（只能从以下选项中选择一个）：\n{name_list}\n\n"
        f"合同首部正文：\n{preamble}\n\n"
        "请只返回合同类型名称，不要返回任何其他内容。"
    )


def _call_llm(prompt: str, settings) -> str | None:
    """调用 LLM API。"""
    timeout = getattr(settings, 'llm_classify_timeout', 10)
    try:
        response = httpx.post(
            settings.llm_api_url,
            headers={
                "Authorization": f"Bearer {settings.llm_api_key}",
                "Content-Type": "application/json",
            },
            json={
                "model": settings.llm_model_name,
                "messages": [{"role": "user", "content": prompt}],
                "temperature": 0,
            },
            timeout=timeout,
        )
        response.raise_for_status()
        data = response.json()
        return data["choices"][0]["message"]["content"]
    except Exception:
        _logger.exception("LLM contract type classification failed")
        return None


def _confidence_from_response(raw: str, matched: str) -> int:
    """根据 LLM 原始返回估算置信度。
    精确匹配置信度最高，包含匹配次之。"""
    if raw == matched:
        return 96
    if raw.startswith(matched):
        return 85
    return 70
```

### 1.3 合同首部（preamble）说明

文档解析后 `contract_structure` 树中，`kind` 为 `preamble` 的节点代表合同首部。此节点由 `contract_structure_parser.py` 中 `preamble_node_payload()` 创建，包含根节点直属正文（合同标题、甲乙方信息等条款正式开始前的内容）。

若解析结果中不存在 `preamble` 节点（如 pymupdf4llm 引擎），降级使用根节点直属 `content[]`。

## 二、修改 `api/contracts.py`

### 2.1 上传端点集成

在 `upload_contract()` 函数中，解析完成后插入 LLM 分类：

```python
@router.post("/upload")
async def upload_contract(file: UploadFile | None = File(default=None)):
    # ... 现有上传 + 解析逻辑 ...

    # LLM 合同类型识别（新增）
    detected_type = None
    match_confidence = 50
    try:
        from src.services.llm_classifier import classify_contract_type
        llm_result = await asyncio.to_thread(
            classify_contract_type, result.metadata.id
        )
        if llm_result:
            detected_type, match_confidence = llm_result
            # 更新 DB 中的合同类型
            repository.update_by_id(
                contract_id,
                {"contract_type": detected_type},
            )
    except Exception:
        pass  # 降级

    if detected_type is None:
        # 降级：使用原有的关键词规则结果
        detected_type = record.get("contract_type") or "未分类"

    return flat_object({
        "id": str(record["id"]),
        "name": record.get("contract_name") or result.metadata.original_name,
        "detectedType": detected_type,
        "matchConfidence": match_confidence,
        "fileId": result.metadata.id,
        "enableStructureEditor": settings.enable_structure_editor,
    })
```

### 2.2 关键点

- 使用 `asyncio.to_thread()` 将同步 httpx 调用放入线程池，避免阻塞事件循环
- LLM 调用失败不抛异常，静默降级
- LLM 推断出的类型**写回 DB**（更新 `contracts.contract_type` 列），使后续查询和历史列表页也能看到

## 三、数据流

```
上传合同
  → file_storage.save_upload()
  → document_tasks.parse_uploaded_document()  [异步线程]
  → contract_extractor.extract_contract_meta() [关键词规则]
  → contract_store.upsert_by_file_id()
  → llm_classifier.classify_contract_type()     [新增：LLM 推断]
       ├─ load_document_json() → 提取 preamble.content[]
       ├─ contract_type_repository.find_all({enabled:1}) → 类型名称清单
       ├─ _build_prompt(首部正文, 清单)
       ├─ httpx.post(LLM_API_URL) → 返回类型名称
       └─ 校验是否在清单中
  → 写回 DB 覆盖 contract_type
  → 返回 { detectedType, matchConfidence }
```

## 四、前端影响

**无前端改动**。

`POST /api/contracts/upload` 返回的 `detectedType` 和 `matchConfidence` 字段不变。  
`ReviewWizard.vue` 中 `reviewStore.detectedType` 已是响应中的值，`StepSetup.vue` 的 `el-select` 默认选中该值。

唯一变化：`detectedType` 的值从"关键词规则推断"变为"LLM 推断"，更准确。
