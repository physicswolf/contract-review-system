# 审查点编辑表单 — 前后端数据对接设计

**日期**：2026-06-29  
**关联页面**：`frontend/src/views/config/PointEdit.vue`  
**关联后端**：`api/points.py` / `api/adapters.py` / `services/audit_point_store.py` / `api/contracts.py`

---

## 一、问题诊断

PointEdit.vue 的表单包含 5 个数据区域，但当前 `save()` 只提交了 `{name, dimension, desc}` 三个字段，导致用户在「风险点信息」「缺省审查结果」「审查示例」区域填写的数据在保存时**被丢弃**。同时 `onMounted` 也只恢复了这三个字段，编辑已有审查点时这些区域无法回显。

此外，前端表单的 JSON 字段命名与后端种子数据/适配器中的命名不一致。

### 1.1 前端表单结构

```javascript
const form = reactive({
  id: '',
  name: '',          // → DB `name` VARCHAR
  dimension: '',     // → DB `dim_id` FK (通过名称查找)
  note: '',          // → DB `instruction` TEXT
  desc: '',          // → DB `desc` VARCHAR
  risks: [           // → DB `risk_points` JSON
    { title: '', clauseNote: '', low: '', high: '' }
  ],
  def: {             // → DB `default_result` JSON
    clause: '', model: '', overview: '', solution: ''
  },
  examples: [        // → DB `examples` JSON
    { clause: '', model: '', overview: '', solution: '' }
  ],
})
```

### 1.2 当前数据流断点

```
[前端 form]                           [后端 DB]
   name    ────────── ✅ ──────────►  name
   dimension ──────── ✅ ──────────►  dim_id (via lookup)
   desc    ────────── ✅ ──────────►  desc
   note    ────────── ❌ 未发送 ────►  instruction
   risks   ────────── ❌ 未发送 ────►  risk_points
   def     ────────── ❌ 未发送 ────►  default_result
   examples ───────── ❌ 未发送 ────►  examples
```

---

## 二、设计决策

### 2.1 JSON 字段命名：以前端为规范

JSON 列直接存储与前端表单一致的字段名，**不使用中间翻译层**。原因：
- 前端是按 Figma 原型开发的，是系统的视觉/交互基准
- JSON 列无 schema 约束，可直接存储任意结构
- 消除翻译层降低维护成本

### 2.2 规范后的 JSON 结构

#### `risk_points`（对应前端 `risks`）

```json
[
  {
    "title": "主体信息缺失",
    "clauseNote": "甲方信息不完整将导致责任主体难以确认",
    "low": "主体信息表述不完整，建议补充核对",
    "high": "主体信息或授权材料缺失将导致责任主体难以确认"
  }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 风险点名称 |
| `clauseNote` | string | 条文说明 |
| `low` | string | 低风险说明 |
| `high` | string | 高风险说明 |

#### `default_result`（对应前端 `def`）

```json
{
  "clause": "甲方：华城数字建设有限公司",
  "model": "主体信息缺失风险模型",
  "overview": "主体信息或签署授权不完整...",
  "solution": "补充双方主体信息、统一社会信用代码..."
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `clause` | string | 合同原文示例 |
| `model` | string | 风险模型 |
| `overview` | string | 风险说明 |
| `solution` | string | 修改建议 |

#### `examples`（对应前端 `examples`）

```json
[
  {
    "clause": "甲方：（盖章处）",
    "model": "仅留盖章处，未填写具体主体信息",
    "overview": "存在较大法律风险",
    "solution": "补充完整的公司名称、法定地址"
  }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `clause` | string | 合同原文 |
| `model` | string | 风险模型 |
| `overview` | string | 风险说明 |
| `solution` | string | 修改建议 |

## 三、变更清单

### 3.1 `api/adapters.py` — 适配器更新

#### `point_to_frontend(detail=True)` 返回字段对照

当前返回字段需要增加 `note` 映射，其余 JSON 字段直接透传：

```python
def point_to_frontend(record, *, detail=False):
    item = {
        "id": str(record["id"]),
        "name": record.get("name", ""),
        "category": record.get("category", ""),
        "dimension": record.get("dim_name", ""),
        "desc": record.get("desc") or "",
        "note": record.get("instruction") or "",        # 新增：instruction → note
        "riskPoints": int(record.get("risk_points_count") or 0),
        "status": record.get("status", "已启用"),
        "defaultChecked": bool(record.get("default_checked", 1)),
        "updatedAt": _format_dt(record.get("updated_at")),
    }
    if detail:
        item.update({
            "dimId": str(record.get("dim_id") or ""),
            "risks": _json_value(record.get("risk_points"), []),       # key 改为 risks
            "def": _json_value(record.get("default_result"), {}),      # key 改为 def
            "examples": _json_value(record.get("examples"), []),       # 保持不变
        })
    return item
```

#### `point_from_frontend` 输入处理

```python
def point_from_frontend(payload):
    result = {}
    # 标量字段映射
    scalar_map = {
        "name": "name", "category": "category",
        "desc": "desc", "status": "status",
        "defaultChecked": "default_checked",
        "dimId": "dim_id", "dimension": "dimension",
    }
    for fk, dk in scalar_map.items():
        if fk in payload:
            result[dk] = payload[fk]
    
    # instruction 来源：前端传 note 或 instruction
    if "note" in payload:
        result["instruction"] = payload["note"]
    elif "instruction" in payload:
        result["instruction"] = payload["instruction"]

    # JSON 字段：前端传 risks/def/examples，直接透传
    if "risks" in payload:
        result["risk_points"] = payload["risks"]
    if "def" in payload:
        result["default_result"] = payload["def"]
    if "examples" in payload:
        result["examples"] = payload["examples"]
    
    return result
```

### 3.2 `frontend/src/views/config/PointEdit.vue` — 保存与加载修复

#### `save()` 提交全部字段

```javascript
async function save() {
  if (!form.name) {
    ElMessage.warning('请填写审核点名称')
    return
  }
  saving.value = true
  try {
    await savePoint({
      id: isNew.value ? undefined : form.id,
      name: form.name,
      dimension: form.dimension,
      note: form.note,
      desc: form.desc,
      risks: form.risks.filter(r => r.title),    // 过滤空行
      def: form.def,
      examples: form.examples.filter(e => e.clause || e.overview || e.solution),  // 过滤空行
    })
    ElMessage.success('保存成功')
    router.push({ name: 'config' })
  } finally {
    saving.value = false
  }
}
```

#### `onMounted` 恢复全部字段

```javascript
onMounted(async () => {
  dims.value = await listDimensions()
  if (!isNew.value) {
    loading.value = true
    try {
      const p = await getPoint(route.params.id)
      if (p) {
        form.id = p.id
        form.name = p.name || ''
        form.dimension = p.dimension || ''
        form.note = p.note || ''
        form.desc = p.desc || ''
        form.risks = (p.risks && p.risks.length) ? p.risks : [{ title: '', clauseNote: '', low: '', high: '' }]
        form.def = p.def || { clause: '', model: '', overview: '', solution: '' }
        form.examples = (p.examples && p.examples.length) ? p.examples : [{ clause: '', model: '', overview: '', solution: '' }]
      }
    } finally {
      loading.value = false
    }
  }
})
```

### 3.4 `services/audit_point_store.py` — 降级默认值更新

`_fallback_json()` 生成的默认 JSON 结构改为前端格式：

```python
def _fallback_json(key: str, payload: dict[str, Any]) -> Any:
    name = str(payload.get("name") or "审查点")
    desc = str(payload.get("desc") or "")
    if key == "risk_points":
        return [{
            "title": name,
            "clauseNote": "",
            "high": desc or "存在较高合同风险",
            "low": "存在一般合同风险",
        }]
    if key == "examples":
        return []
    # default_result
    return {
        "clause": "",
        "model": "",
        "overview": desc or f"{name}存在潜在风险，需要进一步核查。",
        "solution": "建议补充明确条款、责任边界和违约后果。",
    }
```

### 3.5 `api/contracts.py` — 审核结果生成适配

`_build_audit_items()` 读取 `default_result` 和 `risk_points` 时的字段名同步更新：

```python
def _build_audit_items(points, original_blocks):
    items = []
    for index, point in enumerate(points):
        default_result = _json_value(point.get("default_result"), {})
        risk_points = _json_value(point.get("risk_points"), [])
        level = _frontend_level(default_result)
        hit_text = _pick_hit_text(original_blocks, point)
        items.append({
            "dim_id": int(point["dim_id"]),
            "dim_name": str(point.get("dim_name") or ""),
            "audit_point_id": int(point["id"]),
            "title": str(default_result.get("overview") or point.get("name") or "合同风险"),
            "risk_level": 1 if level == "major" else 2,
            "risk_summary": str(
                default_result.get("overview")
                or point.get("desc")
                or "该条款存在潜在风险。"
            ),
            "risk_analysis": str(
                _risk_point_analysis(risk_points, level)
                or default_result.get("overview")
                or "建议结合业务背景进一步核查。"
            ),
            "modify_example": str(
                default_result.get("solution")
                or "建议补充明确约定，降低履约和争议风险。"
            ),
            "clause_name": str(
                default_result.get("clause")
                or point.get("name") or ""
            ),
            "hit_text": hit_text,
            "sort_order": index,
        })
    return items
```

关键变化：
- `default_result.get("analysis")` → `default_result.get("overview")`
- `default_result.get("suggestion")` → `default_result.get("solution")`
- `default_result.get("clause")` 直接可用（原来没有此字段，现在有了）
- `risk_points` 的 `high`/`low` 字段名保持不变（原来就用了这个约定）

#### `_frontend_level()` 更新

由于 `default_result` 不再有 `level` 字段，风险等级由 `risk_points` 中的匹配决定：

```python
def _frontend_level(default_result):
    # 如果有 overview 且包含"高风险"/"重大风险"关键词 → major
    overview = str(default_result.get("overview") or "")
    if any(kw in overview for kw in ["高风险", "重大风险", "严重"]):
        return "major"
    return "general"
```

#### `_risk_point_analysis()` 更新

```python
def _risk_point_analysis(risk_points, level):
    if not isinstance(risk_points, list):
        return ""
    key = "high" if level == "major" else "low"
    for item in risk_points:
        if isinstance(item, dict) and item.get(key):
            return str(item[key])
    return ""
```

## 五、数据流验证路径

```
[新建审查点]
  用户填写全部字段 → save() 发送 {name, dimension, note, desc, risks, def, examples}
  → point_from_frontend 映射 → DB INSERT {name, dim_id, instruction, desc, risk_points, default_result, examples}
  → 成功返回 → 跳转列表页

[编辑审查点]
  列表页点击编辑 → getPoint(id) → GET /api/points/:id
  → point_to_frontend(detail=True) 返回 {name, dimension, note, desc, risks, def, examples}
  → onMounted 恢复所有表单字段 → 用户修改 → save() 全量提交

[审核执行 v1]
  发起审核 → 读取 audit_point 记录的 risk_points/default_result/examples
  → _build_audit_items 按新字段名读取 → 生成 audit_result_item
  → GET /contracts/:id/result 返回审核结果
```

## 六、兼容性说明

- 旧种子数据已在数据库中但使用旧 JSON 格式（`{level, analysis, suggestion}` / `{name, highStd, lowStd}`），需要执行迁移脚本重新 INSERT IGNORE（先物理删除旧审查点种子数据，再 INSERT IGNORE 新格式）
- 或者：后端 `_json_value` 已经兼容 dict/list 原生类型和 JSON 字符串，新老格式不会导致解析异常；只是字段名不同会导致 `_build_audit_items` 读不到数据（返回空字符串），不影响系统稳定性
- 建议：重新执行 `004_seed_default_data.sql`（先 `TRUNCATE audit_point` 再 `INSERT IGNORE`，保留用户手动新增的数据则需 UPDATE 迁移）
