## 三、设计说明书对照

| 设计要求 | 实现 | 状态 |
|----------|------|------|
| DOCX → python-docx 段落提取 + 编号 | `_extract_markdown_lines()` | ✅ |
| DOCX → document.json → texts+bbox 提取 | `_load_document_units()` | ✅ |
| 文本匹配算法（精确→前缀→loose） | `_find_line_start()` 三级匹配 | ✅ |
| 同页 bbox 合并 | `_union_block_bbox()` | ✅ 且 `_build_bbox` 只取同页 units |
| PDF 直接提取 | `_build_pdf_blocks()` | ✅ |
| 解析完成后调用 | `_build_blocks_artifact()` 在成功分支 | ✅ |
| 失败不阻断 | `except Exception: logger` | ✅ |
| blocks.json 原子写入 | `_write_json_atomically` | ✅ |

**额外亮点**（超出设计说明书）：
- `_prepend_label()` 去重逻辑：避免 python-docx 已将编号写入段落文本时产生双重编号
- `_find_toc_pages()` 过滤目录页：防止目录中的章节标题重复匹配
- `_build_bbox()` 限制同页 units 合并：跨页时不混合 bbox
- `_shortened_candidates()` 短化匹配：处理标点符号导致的文本断裂
