- 上传端点包装了解析管线，同步等待任务完成
- 响应包含 `enableStructureEditor` 字段
- 删除操作返回 `{ok:true}`

**🟡 问题 7.1：上传端点使用同步等待（`_wait_for_task`）**

`_wait_for_task()` 以 0.1s 间隔轮询最多 90 秒。对于大文件（如扫描件 PDF），Docling 解析可能需要超过 90 秒，此时会超时失败。原始后端设计是异步模式（前端轮询 `/files/tasks/{id}`）。

**建议**：在响应注释中标注超时限制，或在 v2 改为返回 task ID 让前端异步轮询。

### 模块 9：main.py 整合 🔴 1 个严重问题

路由注册结构正确，删除了旧 web/compat 路由。

**🔴 问题 9.1：`audit_router` 注册了空路由**

```python
app.include_router(audit_router, prefix=f"{settings.api_prefix}/audit", tags=["审核"])
```

但 `api/audit.py` 中的 router 是空的：
```python
router = APIRouter()  # 无任何端点
```

这会导致 `/api/audit` 前缀下没有任何可用端点。审核相关的端点（`/contracts/:id/audit` 和 `/contracts/:id/result`）实际上在 `contracts_router` 中实现，所以功能不受影响。但多余的空路由注册会产生一个无用的 `/api/audit` 前缀空间，以及 `/docs` 中会出现一个空的 "审核" 标签组。

**修复**：从 `main.py` 中删除 `audit_router` 的注册及其 import，或从 `api/audit.py` 中移除。

### 模块 10：前端适配 🔴 1 个严重问题

`config.js` 正确切换为环境变量控制。`point.service.js` 正确改为 `/dimensions/names`。

**🔴 问题 10.1：缺少 `structureEditor` 路由（严重）**

`ReviewWizard.vue` 第 69 行：
```javascript
if (res.enableStructureEditor && ENABLE_STRUCTURE_EDITOR) {
    router.push({ name: 'structureEditor', params: { fileId: res.fileId } })
    return
}
```

但 `router/index.js` 中**不存在**名为 `structureEditor` 的路由。当前因为 `ENABLE_STRUCTURE_EDITOR` 默认为 `false` 不会触发，但一旦设为 `true`，前端会抛出路由错误导致页面白屏。

**修复**：在 `router/index.js` 中添加结构编辑路由：
```javascript
{ path: 'structure-editor/:fileId', name: 'structureEditor', component: () => import('../views/editor/StructureEditor.vue') }
```

由于结构编辑页面的 Vue 组件目前不存在，至少应添加一个占位组件或使用条件注释标注这是一个待实现的页面。短期内更安全的做法是在条件判断中添加 `// TODO: 待结构编辑页面就绪后启用` 注释，并确保即使触发也降级到 `step.value = 2`。
