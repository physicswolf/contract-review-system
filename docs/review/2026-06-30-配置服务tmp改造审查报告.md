## 三、发现的问题

### 🟡 中等（3 个）

**1. `api/audit.py` 残留空文件**

文件存在但 `main.py` 不注册。建议删除。

**2. `009_seed_tmp_config_data.sql` 使用 `INSERT` 非 `INSERT IGNORE`**

`008` 已先 DROP 表所以首次安全。但若单独重跑 009 会因主键冲突失败。建议改为 `INSERT IGNORE` 或在文件顶部加 `TRUNCATE`。

**3. `/api/points` 旧端点与 `/api/audit-points` 并存**

`main.py` 同时注册了 `points_router`（旧扁平格式）和 `audit_points_router`（tmp 格式）。前端已切换为 `/audit-points`，但审核向导的 `StepSetup.vue` 中 `listPoints()` → `point.service.js` → `/audit-points`，确认路径正确。旧 `/api/points` 无前端调用，可作为内部兼容保留或后续移除。

### 🟢 轻微（2 个）

**4. `_risk_point_analysis` 不支持旧格式 `high`/`low`**

只读取 `highStd`/`lowStd`（tmp 格式），若 DB 中仍存有旧格式 `{title, high, low}` 数据会返回空。由于 DB 已重建为 tmp 格式，实际无影响。

**5. 测试文件缺失**

后端测试目录 `tests/` 不存在，无法执行回归验证。需从原始目录恢复。

## 六、设计文档与 AGENTS.md 冲突处理

### 6.1 冲突描述

设计说明书 `后端配置服务对齐tmp规范改造设计说明书.md` 第六章要求修改 `frontend/src/api/mock/db.js`，将 mock 数据字段对齐 tmp 格式。

但 `contract-review-system/AGENTS.md` 第 7 节「禁止事项」第 3 条明确规定：

> ❌ 不要修改 `frontend/src/api/mock/db.js`

### 6.2 冲突分析

| 来源 | 立场 | 理由 |
|------|------|------|
| 设计说明书 | 修改 mock/db.js | 使 mock 数据与 tmp 后端格式一致，方便 `USE_MOCK=true` 时调试 |
| AGENTS.md | 禁止修改 | mock/db.js 是前端 UI 的**数据契约基准**，修改它会导致"mock 驱动开发"的参照物失真。真正的适配应在后端和 service adapter 层完成 |

两者看似矛盾，实则服务不同目标：
- 设计说明书追求 **mock 与后端一致**（开发调试便利）
- AGENTS.md 追求 **mock 作为不可变契约**（保证前端 UI 的独立可用性）

### 6.3 裁决

**AGENTS.md 优先级高于设计说明书**。理由：

1. AGENTS.md 是项目级治理文件，约束所有开发行为；设计说明书是本轮迭代的工作文档
2. 前端 service 层（`point.service.js`、`dimension.service.js` 等）已经通过 `adaptPoint()`/`adapt()` 函数完成了"后端格式 → 前端格式"的转换。Mock 数据代表的是**前端格式**（adapter 输出端），不需要与后端一致
3. mock/db.js 保持原样恰好验证了 adapter 层的正确性：`USE_MOCK=true` 时 adapter 接收前端格式数据直接透传，`USE_MOCK=false` 时 adapter 接收后端格式数据并转换
4. 本轮未修改此文件的决定是正确的

### 6.4 修正措施

1. **设计说明书**中「前端 mock 数据更新」一节标注为 **已撤销**（不执行）
2. **AGENTS.md** 中禁止修改 mock/db.js 的条款**继续有效**
3. 未来如果前端 UI 需求变更导致 mock 数据结构需要调整，应先修改 AGENTS.md 的相关条款，再修改 mock 数据
