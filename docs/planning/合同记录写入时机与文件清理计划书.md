## 一、现状与问题

### 1.1 当前流程

```
上传文件 → 解析完成 → 立即写入 contracts 表（2处：document_tasks + upload端点）
         → 返回 { id(contract_id), fileId, ... }
         
结构编辑（可选）→ 点击"返回上传" → 仅跳转回 Step1，文件未删除

审核设置 → 点击"开始分析" → POST /contracts/{contract_id}/audit（contract 已存在）
```

### 1.2 问题

| 问题 | 说明 |
|------|------|
| 上传即写库 | 用户可能上传后不满意、重新上传、或中途离开，contracts 表残留无用记录 |
| 返回/重传不清理 | 结构编辑页"返回上传"和审核设置页"重新上传"后，已上传文件和解析目录仍占磁盘 |
| 已删除的合同 ID 无法复用 | `reviewStore.contractId` 来自已删除的记录，`startAudit` 会 404 |

## 二、目标流程

```
上传文件 → 保存文件 + 解析 → 返回 { fileId, name, detectedType, matchConfidence }
          ↓ 不写 contracts 表
          
结构编辑（可选）
  ├─ 点"返回上传" → DELETE /api/files/{fileId}/artifacts → 清理文件 → 回到 Step1
  └─ 点"保存" → 继续

审核设置
  ├─ 点"重新上传" → DELETE /api/files/{fileId}/artifacts → 清理文件 → 回到 Step1
  └─ 点"开始分析" → POST /api/contracts/audit { fileId, role, points, contractType }
                   → 此时才写入 contracts 表
                   → 创建审核任务 → 返回 { id(contract_id), status }
```
