## 一、背景

原型 `design/AI智契.html` 中合同类型管理采用 **表格列表 + 行内操作** 的模式，与审核点管理 Tab 保持一致。当前 `ConfigCenter.vue` 中合同类型 Tab 使用的是左列表+右编辑器的分栏布局，不符合原型设计模式。

## 二、当前状态 vs 目标

| 对比 | 当前 | 目标（原型模式） |
|------|------|-----------------|
| 列表形式 | 左侧 `type-aside` 卡片列表 | 表格（`el-table`），含列：名称、立场、描述、关联审查点数、状态、操作 |
| 搜索 | 无 | 搜索框 + 检索/重置按钮 |
| 状态展示 | 无 | `el-switch` 启停开关 |
| 新建 | 底部按钮 | 顶栏 `＋ 新建合同类型` 按钮 |
| 编辑入口 | 点击左侧卡片 | 表格行末 `配置` + `关联审核点` 按钮 |
| 编辑方式 | 右侧面板内联 | 弹层（`el-dialog`） |
| 统计 | 无 | 合同类型总数 + 已启用数 |
