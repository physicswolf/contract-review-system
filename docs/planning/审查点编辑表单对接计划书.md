# 审查点编辑表单对接计划书

**项目**：contract-review-system  
**日期**：2026-06-29  
**版本**：v1.0  
**状态**：已完成  
**对应设计说明书**：`docs/develop/审查点编辑表单-前后端数据对接设计.md`

---

## 一、问题诊断

PointEdit.vue 是配置中心中审查点的编辑页面，包含 5 个数据区域：

| 区域 | 表单字段 | 对应 DB 列 | 类型 |
|------|---------|-----------|------|
| 基本信息 | `name`, `dimension`, `note`, `desc` | `name`, `dim_id`, `instruction`, `desc` | 标量 |
| 风险点信息 | `risks[]` — `{title, clauseNote, low, high}` | `risk_points` | JSON |
| 缺省审查结果 | `def` — `{clause, model, overview, solution}` | `default_result` | JSON |
| 审查示例 | `examples[]` — `{clause, model, overview, solution}` | `examples` | JSON |

**当前问题**：

1. **保存数据丢失**：`save()` 只提交 `{name, dimension, desc}`，`note`、`risks`、`def`、`examples` 未发送到后端
2. **编辑回显不完整**：`onMounted` 只恢复 `{id, name, dimension, desc}`，其余字段不加载
3. **JSON 字段命名不一致**：前端表单与后端种子数据的 JSON 内部字段名不同

### 数据流断点

```
[前端 form]                         [后端 DB]
   name    ─── ✅ ──►  name
   dimension ── ✅ ──►  dim_id (via lookup)
   desc    ─── ✅ ──►  desc
   note    ─── ❌ 未发送 ──►  instruction
   risks   ─── ❌ 未发送 ──►  risk_points
   def     ─── ❌ 未发送 ──►  default_result
   examples ── ❌ 未发送 ──►  examples
```

---

## 二、设计决策

### 2.1 JSON 字段命名：以前端为规范

JSON 列直接存储与前端表单一致的字段名，不使用中间翻译层。

### 2.2 规范后的 JSON 结构

#### `risk_points`（对应前端 `risks`）

```json
[{ "title": "风险点名称", "clauseNote": "条文说明", "low": "低风险说明", "high": "高风险说明" }]
```

#### `default_result`（对应前端 `def`）

```json
{ "clause": "合同原文", "model": "风险模型", "overview": "风险说明", "solution": "修改建议" }
```

#### `examples`（对应前端 `examples`）

```json
[{ "clause": "合同原文", "model": "风险模型", "overview": "风险说明", "solution": "修改建议" }]
```

## 四、变更详细说明

### 4.1 适配器 (`api/adapters.py`)

**`point_to_frontend(detail=True)`** 返回结构变更：

| 变更前 | 变更后 | 说明 |
|--------|--------|------|
| 无 `note` 字段 | 新增 `note: record.instruction` | 前端表单的审查点说明 |
| `risk_points` 键 | → `risks` 键 | 与前端 form.risks 对应 |
| `defaultResult` 键 | → `def` 键 | 与前端 form.def 对应 |
| `riskPointsDetail` | 删除 | 改用 `risks` |
| `instruction` | 删除 | 改用 `note` |

**`point_from_frontend`** 输入处理变更：

| 变更前 | 变更后 | 说明 |
|--------|--------|------|
| 无 `note` 处理 | 新增 `note→instruction` 映射 | 前端 note 写入 DB instruction |
| `riskPointsDetail` 优先 | `risks` 优先 | 统一为前端字段名 |
| `defaultResult` 优先 | `def` 优先，`defaultResult` 降级兼容 | 兼容前端新旧版本 |

### 4.2 前端 (`PointEdit.vue`)

**`save()` 变更**：
```javascript
// 变更前：仅3个字段
await savePoint({ id, name: form.name, dimension: form.dimension, desc: form.desc })

// 变更后：全部字段，过滤空行
await savePoint({
  id: isNew.value ? undefined : form.id,
  name: form.name,
  dimension: form.dimension,
  note: form.note,
  desc: form.desc,
  risks: form.risks.filter(r => r.title),
  def: form.def,
  examples: form.examples.filter(e => e.clause || e.overview || e.solution),
})
```

**`onMounted` 变更**：
```javascript
// 变更前：仅3个字段
Object.assign(form, { id: p.id, name: p.name, dimension: p.dimension, desc: p.desc })

// 变更后：全部字段
form.id = p.id; form.name = p.name || ''; form.dimension = p.dimension || ''
form.note = p.note || ''; form.desc = p.desc || ''
form.risks = (p.risks?.length) ? p.risks : [{ title: '', clauseNote: '', low: '', high: '' }]
form.def = p.def || { clause: '', model: '', overview: '', solution: '' }
form.examples = (p.examples?.length) ? p.examples : [{ clause: '', model: '', overview: '', solution: '' }]
```

### 4.3 种子数据 (`004_seed_default_data.sql`)

- `risk_points` JSON：每条增加 `"clauseNote":""` 字段
- `default_result` JSON：`{"level":"高风险","analysis":"...","suggestion":"..."}` → `{"clause":"","model":"","overview":"...","solution":"..."}`

### 4.4 已部署数据库升级 (`007_update_point_json_format.sql`)

```sql
DELETE FROM audit_point WHERE id <= 18;
INSERT INTO audit_point (...) VALUES ...  -- 含新格式 JSON
```

### 4.5 降级默认值 (`audit_point_store.py`)

`_fallback_json()` 生成的默认 JSON 结构改为前端格式：
- `risk_points` 默认值：`[{title, clauseNote:"", high:"...", low:"..."}]`
- `default_result` 默认值：`{clause:"", model:"", overview:"...", solution:"..."}`

### 4.6 审核引擎 (`api/contracts.py`)

`_build_audit_items()` 读取 `default_result` 时的字段名：
- `default_result.analysis` → `default_result.overview`
- `default_result.suggestion` → `default_result.solution`
- `default_result.clause` 直接可用（新增字段）

`_frontend_level()` 风险等级判断：
- 优先检查 `default_result.level`（兼容旧格式）
- 降级：overview 含"高风险/重大风险/严重/可能无效"关键词 → major

## 五、验证路径

```
[新建审查点]
  填写全部字段 → save() 全量提交 → DB 写入全部列 → 列表页显示 ✅

[编辑审查点]
  列表页点击编辑 → GET /points/:id → 返回 {risks, def, examples}
  → onMounted 恢复全部表单字段 → 用户修改 → save() 全量提交 ✅

[审核执行]
  发起审核 → 读取 risk_points + default_result
  → _build_audit_items 按新字段名读取 → 生成 audit_result_item ✅
```
