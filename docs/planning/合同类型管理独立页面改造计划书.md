## 一、背景

当前 `ConfigCenter.vue` 中合同类型的新建和编辑使用 `el-dialog` 弹层。按原型设计模式，新建和编辑应为**独立导航页面**，含"返回合同类型管理"按钮。

## 二、现状

| 项目 | 现状 | 目标 |
|------|------|------|
| 新建入口 | `ConfigCenter.vue` 中 `el-dialog` 弹层 | 路由跳转至 `/config/contract-types/new` |
| 编辑入口 | `ConfigCenter.vue` 中 `el-dialog` 弹层 | 路由跳转至 `/config/contract-types/:id/edit` |
| 返回按钮 | 无 | 页面顶部"返回合同类型管理"按钮 |

## 三、后端分析

后端 `api/contract_types.py` 已有完整 CRUD 端点：

| Method | Path | 用途 | 状态 |
|--------|------|------|------|
| GET | `/api/contract-types` | 列表（含 `linkedPointCount`） | ✅ 已有 |
| GET | `/api/contract-types/:id` | 详情（含 `linkedAuditPoints`） | ✅ 已有 |
| POST | `/api/contract-types` | 创建 | ✅ 已有 |
| PUT | `/api/contract-types/:id` | 更新 | ✅ 已有 |
| DELETE | `/api/contract-types/:id` | 软删除 | ✅ 已有 |

**后端无需新增端点**。仅需前端新建页面、路由、修改 ConfigCenter 按钮为导航跳转。
