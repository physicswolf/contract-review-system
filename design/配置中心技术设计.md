# 配置中心技术设计文档（简化版）

## 1. 概要

### 1.1 背景

配置中心是 AI智契 合同审核系统的规则管理工作台。管理员通过「规则分类 → 风险点」两级结构定义审核标准，AI 引擎在审核合同时加载这些规则作为判断依据。

### 1.2 功能范围

| 模块 | 功能 |
|------|------|
| 规则分类管理 | 创建/编辑/删除分类，关联合同类型（含自定义类型），控制启停 |
| 风险点管理 | 在分类下增删改查风险点，支持跨分类移动和排序 |
| 审核规则管理 | 分页检索规则，按名称/条款类型搜索 |
| 规则编辑 | 配置规则基础信息、多风险点（含原文/风险说明/风险分析）、示例及默认输出 |

### 1.3 设计原则

- **架构极简**：MySQL 单库承载所有数据，无 Redis、无消息队列
- **暂不考虑**：用户登录认证、多租户、权限控制（后续扩展预留字段即可）

## 3. 数据库设计

### 3.1 规则分类表 `rule_category`

```sql
CREATE TABLE rule_category (
  id           BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  name         VARCHAR(100)    NOT NULL COMMENT '分类名称，如：基础核查、法务风险',
  description  TEXT            COMMENT '分类说明',
  status       TINYINT         NOT NULL DEFAULT 1 COMMENT '1启用 0停用',
  sort_order   INT             NOT NULL DEFAULT 0,
  created_at   DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at   DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at   DATETIME        DEFAULT NULL,
  PRIMARY KEY (id),
  KEY idx_status (status, deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.2 分类关联合同类型表 `category_contract_type`

```sql
CREATE TABLE category_contract_type (
  id            BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  category_id   BIGINT UNSIGNED NOT NULL,
  contract_type VARCHAR(50)     NOT NULL COMMENT '合同类型名称',
  PRIMARY KEY (id),
  UNIQUE KEY uk_category_type (category_id, contract_type),
  KEY idx_category (category_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.3 自定义合同类型表 `custom_contract_type`

```sql
CREATE TABLE custom_contract_type (
  id         BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  name       VARCHAR(100)    NOT NULL COMMENT '自定义类型名称',
  created_at DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at DATETIME        DEFAULT NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uk_name (name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.4 风险点表 `risk_point`

```sql
CREATE TABLE risk_point (
  id              BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  category_id     BIGINT UNSIGNED NOT NULL COMMENT '所属规则分类',
  title           VARCHAR(200)    NOT NULL COMMENT '风险点标题',
  risk_level      TINYINT         NOT NULL COMMENT '1高风险 2中风险 3低风险',
  original_clause TEXT            NOT NULL COMMENT '合同原文/典型表述',
  risk_desc       TEXT            NOT NULL COMMENT '风险说明',
  risk_analysis   TEXT            COMMENT '风险分析',
  sort_order      INT             NOT NULL DEFAULT 0,
  created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at      DATETIME        DEFAULT NULL,
  PRIMARY KEY (id),
  KEY idx_category (category_id, deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.5 审核规则表 `audit_rule`

> 规则是可独立配置的审核单元，直接驱动 AI 引擎，可关联若干风险点。

```sql
CREATE TABLE audit_rule (
  id                   BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  name                 VARCHAR(200)    NOT NULL COMMENT '规则名称',
  clause_type          VARCHAR(50)     NOT NULL COMMENT '条款类型',
  risk_level           TINYINT         NOT NULL COMMENT '1高风险 2中风险 3低风险',
  contract_scope       VARCHAR(50)     NOT NULL COMMENT '适用合同类型',
  description          TEXT            COMMENT '规则描述',
  -- 示例
  example_clause       TEXT            COMMENT '合同原文示例',
  example_risk_level   TINYINT         COMMENT '示例风险等级',
  example_suggestion   TEXT            COMMENT '修改建议示例',
  -- 默认审核输出（风险点未单独配置时使用）
  default_risk_level   TINYINT         DEFAULT 2,
  default_risk_analysis TEXT,
  default_modify_example TEXT,
  status               TINYINT         NOT NULL DEFAULT 1 COMMENT '1启用 0停用',
  created_at           DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at           DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at           DATETIME        DEFAULT NULL,
  PRIMARY KEY (id),
  KEY idx_status (status, deleted_at),
  KEY idx_clause_type (clause_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.6 规则-风险点关联表 `rule_risk_point`

```sql
CREATE TABLE rule_risk_point (
  id            BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  rule_id       BIGINT UNSIGNED NOT NULL,
  risk_point_id BIGINT UNSIGNED NOT NULL,
  sort_order    INT             NOT NULL DEFAULT 0,
  PRIMARY KEY (id),
  UNIQUE KEY uk_rule_point (rule_id, risk_point_id),
  KEY idx_rule (rule_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 4. 表关系总览

```
rule_category (1)
    │
    ├── category_contract_type (N)   ← 分类关联的合同类型
    └── risk_point (N)               ← 分类下的风险点

audit_rule (1)
    └── rule_risk_point (N)          ← 规则引用的风险点

custom_contract_type                 ← 企业自定义合同类型（独立）
```

## 5. API 设计

> 基础路径：`/api/v1/config`

### 5.1 规则分类

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/categories` | 分页查询分类列表 |
| POST | `/categories` | 创建分类 |
| PUT | `/categories/:id` | 更新分类（含启停） |
| DELETE | `/categories/:id` | 删除分类（软删除，联级删除风险点） |

#### GET /categories

Query: `keyword`（名称搜索）、`page`（默认1）、`page_size`（默认10）

```json
{
  "code": 0,
  "data": {
    "total": 12,
    "list": [
      {
        "id": 1,
        "name": "交易条款核查",
        "description": "识别付款、交付、验收等风险",
        "contract_types": ["采购合同", "服务合同"],
        "risk_point_count": 8,
        "status": 1,
        "updated_at": "2026-06-19T09:30:00+08:00"
      }
    ]
  }
}
```

#### POST /categories

```json
// 请求
{
  "name": "交易条款核查",
  "description": "识别付款、交付、验收等风险",
  "contract_types": ["采购合同", "服务合同"]
}
// 响应
{ "code": 0, "data": { "id": 1 } }
```

## 6. 核心逻辑说明

### 6.1 软删除

所有删除操作写 `deleted_at = NOW()`，查询统一加 `deleted_at IS NULL`。
分类删除时同步软删除其下全部风险点，但不自动删除关联该分类风险点的规则（需人工处理）。

### 6.2 系统内置合同类型

在代码层面维护一份静态列表（采购合同、服务合同、销售合同、租赁合同、合作协议、通用合同），不入库。GET /contract-types 返回时合并 `custom_contract_type` 表中的自定义类型。
